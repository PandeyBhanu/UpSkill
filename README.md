# PlacementDuel

A full-stack competitive placement preparation application where two users (duel partners) can compete head-to-head in AI-generated weekly tests.

## Features

- **Partner System**: Link with a study partner to compete in weekly tests
- **Weekly Topics**: Submit study topics each week, which are used to generate personalized tests
- **AI Test Generation**: Tests generated by Google Gemini 2.0 Flash based on both partners' topics
- **Strict Interview Mode**: Timed sections with auto-submission, no going back to previous sections
- **Code Execution**: Built-in Monaco Editor with Judge0 API for running code (Python, Java, C++)
- **AI Evaluation**: Answers evaluated by Groq's Llama 3.3 model with detailed feedback
- **Head-to-Head Results**: Compare scores with your partner section by section
- **Analytics Dashboard**: Track performance trends, identify weak zones with AI insights
- **Achievement Badges**: Earn badges like "First Win 🥇", "Perfect HR ✨", "Code Warrior 💻", "Consistent Streak 🔥"

## Tech Stack

- **Frontend**: Next.js 14 (App Router), React, TypeScript
- **Styling**: Tailwind CSS v4
- **Database/Auth**: Supabase (PostgreSQL + Auth)
- **AI Services**:
  - Google Gemini 2.0 Flash (test generation)
  - Groq Llama 3.3 (answer evaluation)
- **Code Execution**: Piston API (free public instance)
- **Charts**: Recharts
- **Icons**: Lucide React
- **Code Editor**: Monaco Editor

## Prerequisites

Before running this application, you need to set up accounts and obtain API keys for the following services (all free tiers):

1. **Supabase** ([https://supabase.com](https://supabase.com))
   - Create a new project
   - Get your project URL and anon key

2. **Google Gemini API** ([https://makersuite.google.com](https://makersuite.google.com))
   - Create an API key

3. **Groq API** ([https://groq.com](https://groq.com))
   - Create an account and get an API key

4. **Piston API** (for code execution)
   - Uses free public instance (no API key required)
   - https://emkc.org/api/v2/piston/execute

## Setup Instructions

### 1. Install Dependencies

```bash
npm install
```

### 2. Configure Environment Variables

Create a `.env.local` file in the project root:

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key

# Google Gemini API
GEMINI_API_KEY=your_gemini_api_key

# Groq API
GROQ_API_KEY=your_groq_api_key

# Piston API (code execution - no API key needed for public instance)
```

### 3. Set Up Supabase Database

1. Go to your Supabase project dashboard
2. Navigate to the SQL Editor
3. Run the SQL schema from `supabase/schema.sql`:

```sql
-- Enable UUID extension
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- Users table
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  email TEXT UNIQUE NOT NULL,
  name TEXT NOT NULL,
  pair_id UUID REFERENCES pairs(id),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Pairs table
CREATE TABLE pairs (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user1_id UUID REFERENCES users(id) ON DELETE CASCADE,
  user2_id UUID REFERENCES users(id) ON DELETE CASCADE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Weekly topics table
CREATE TABLE weekly_topics (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  week_start_date DATE NOT NULL,
  topics_text TEXT NOT NULL,
  tags TEXT[],
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE(user_id, week_start_date)
);

-- Tests table
CREATE TABLE tests (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  pair_id UUID REFERENCES pairs(id) ON DELETE CASCADE,
  week_start_date DATE NOT NULL,
  generated_questions JSONB NOT NULL,
  status TEXT DEFAULT 'pending',
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE(pair_id, week_start_date)
);

-- Responses table
CREATE TABLE responses (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  test_id UUID REFERENCES tests(id) ON DELETE CASCADE,
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  question_id TEXT NOT NULL,
  answer TEXT NOT NULL,
  submitted_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE(test_id, user_id, question_id)
);

-- Scores table
CREATE TABLE scores (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  test_id UUID REFERENCES tests(id) ON DELETE CASCADE,
  user_id UUID REFERENCES users(id) ON DELETE CASCADE,
  total_score INTEGER NOT NULL,
  section_scores JSONB NOT NULL,
  per_question_feedback JSONB NOT NULL,
  badges TEXT[],
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE(test_id, user_id)
);

-- Indexes for better query performance
CREATE INDEX idx_users_pair_id ON users(pair_id);
CREATE INDEX idx_weekly_topics_user_id ON weekly_topics(user_id);
CREATE INDEX idx_weekly_topics_week_start_date ON weekly_topics(week_start_date);
CREATE INDEX idx_tests_pair_id ON tests(pair_id);
CREATE INDEX idx_tests_week_start_date ON tests(week_start_date);
CREATE INDEX idx_responses_test_id ON responses(test_id);
CREATE INDEX idx_responses_user_id ON responses(user_id);
CREATE INDEX idx_scores_test_id ON scores(test_id);
CREATE INDEX idx_scores_user_id ON scores(user_id);

-- Function to update updated_at timestamp
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- Triggers for updated_at
CREATE TRIGGER update_users_updated_at BEFORE UPDATE ON users
  FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();

CREATE TRIGGER update_pairs_updated_at BEFORE UPDATE ON pairs
  FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();

CREATE TRIGGER update_weekly_topics_updated_at BEFORE UPDATE ON weekly_topics
  FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();

CREATE TRIGGER update_tests_updated_at BEFORE UPDATE ON tests
  FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();

CREATE TRIGGER update_responses_updated_at BEFORE UPDATE ON responses
  FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();

CREATE TRIGGER update_scores_updated_at BEFORE UPDATE ON scores
  FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();
```

4. Enable Email Auth in Supabase:
   - Go to Authentication > Providers
   - Enable Email provider
   - Configure email settings (you can use Supabase's built-in email for testing)

### 4. Run the Development Server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## Usage Guide

### Getting Started

1. **Sign Up**: Create an account with your name, email, and password
2. **Link Partner**: During signup or later, enter your partner's email to link accounts
3. **Submit Topics**: Go to "Weekly Topics" and submit what you studied this week
4. **Generate Test**: Once both partners submit topics, click "Generate Test"
5. **Take Test**: Complete the timed test with 4 sections
6. **View Results**: See your score, feedback, and comparison with your partner
7. **Check Analytics**: View trends and get AI-powered weak zone analysis

### Test Structure

Each test has 4 sections with time limits:

- **CS Fundamentals** (20 minutes): 6 questions (MCQ + short answer)
- **DSA / Coding** (35 minutes): 3 questions (1 MCQ, 2 coding problems)
- **System Design** (20 minutes): 1 open-ended question
- **HR / Behavioural** (10 minutes): 2 situational questions

### Badges

- **First Win 🥇**: Score 80+ on a test
- **Perfect HR ✨**: Score 18+ in HR section (out of 20)
- **Code Warrior 💻**: Score 25+ in DSA section (out of 30)
- **Consistent Streak 🔥**: Score 70+ for 3 consecutive weeks

## Troubleshooting

### Supabase Auth Issues
- Make sure Email provider is enabled in Supabase dashboard
- Check that your anon key is correct
- Verify Row Level Security (RLS) policies if needed

### Gemini API Errors
- Verify your API key is valid
- Check that you have sufficient quota
- Ensure the model name is correct (gemini-2.0-flash-exp)

### Piston API Errors
- Piston uses a free public instance which may be slow or rate-limited
- If execution fails frequently, consider self-hosting Piston
- Check that the language name is correct (python, java, c++)

### Groq API Errors
- Verify your API key
- Check that you have available credits
- Ensure the model name is correct (llama-3.3-70b-versatile)
